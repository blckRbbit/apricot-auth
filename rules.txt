CODE STYLE

- Импорты, как и аннотации, следуют по длине строки по возрастанию, все статические импорты - ниже, после отступа
  в 1 строку;

- Переменные и поля класса объявляются в начале класса/метода;

- Используется отступ в 4 пробела;

- Сущности всегда имеют постфикс Entity в названии класса;

- DTO всегда имеют постфикс DTO в названии класса;

- Если DTO используется для запроса данных, в названии класса должно присутствовать слово Request;

- Если DTO используется для получения ответа от стороннего API, в названии класса должно присутствовать слово Response;

- Для DTO и его полей всегда пишется javadoc с кратким описанием, @JsonProperty - snake case;

- В операторах условий (if, else, switch) можно не использовать фигурные скобки, если код - одна строка, но код пишется
  на другой строке (отступ - 4 пробела);

- В методах используется вертикальный отступ для разделения логических блоков кода (например, инициализация переменных,
  отступ, логический блок кода);

- Оператор var не используется;

- Для локальной разработки каждый создает себе application.yml файл, отнаследованный от application-dev.yml и имеющий
  уникальный постфикс. Файл не пушится в гит;
